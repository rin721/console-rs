# Aoi[葵] 目标证据审计

> 生成时间：2026-06-21
>
> 本文用于 AI 协作者和维护者核对 `/goal` 总目标的完成证据。它不是长期产品规则；长期规则仍以 `AGENTS.md`、`README.md`、`docs/architecture/*`、`docs/api/*` 和项目专属 skill 为准。

## 结论

当前仓库已经具备 Aoi[葵] Rust 化共享产品底座的主干：Cargo workspace、axum/tower HTTP 服务、SQLite 迁移、配置分层、类型化 route registry、OpenAPI 快照、Setup/IAM/System 核心 API、React WebUI、中文文档、根 `AGENTS.md` 和项目专属 skills 都已落地。

尚不能把总目标标记为 100% 完成。原因是部分目标仍是“基础闭环已实现，但生产级外延暂缓”或“需要更强的数据库扩展和最终全链验证”。后续完成验收前，应继续补齐证据，而不是扩大旧架构兼容面。

阶段验收报告见 `docs/ai/stage-acceptance-report.md`。该报告按“已完成内容、进度百分比、剩余任务、删除的旧设计、验证结果、残留风险和下一步”组织当前阶段证据，但不代表最终完成声明。

## 仍需补强的证据

1. 暂缓能力边界：bucket 生命周期/权限策略控制面、外部版本部署执行器、深度主机指标必须保持“真实后端优先”，不能用前端 mock 宣称完成；Prometheus 文本导出已接入真实 `sysinfo` CPU/内存/磁盘/load/network 字段，并由 `metrics-policy` 证明匿名访问不泄露指标正文、授权 scrape 可返回真实指标，但目标环境监控系统配置和更深采集器仍需验收；`scripts/webui-capability-boundary-scan.ps1` 已把前两个暂缓项和 mock/fake UI 能力设为前端运行源码门禁。
2. 外部数据库运行态：PostgreSQL/MySQL 已接入 app runtime，但仍需在目标生产环境完成外部数据库 smoke、部署 preflight、备份/回滚演练、连接池参数压测和目标网络验证。

## 下一步建议

优先补“安全与生产证据缺口”，再补“能力外延”：

1. 在目标服务中用实际 `auth.session_secret` 配置 `observability.prometheus_scrape_token_hash`，raw scrape token 只注入监控系统和验收脚本进程；随后在真实 PostgreSQL/MySQL 服务上运行 `scripts/target-environment-acceptance.ps1 -Driver <driver> -Url <database-url> -BaseUrl <base-url> -ApplyMigrations`，再运行 `scripts/validate-target-acceptance-report.ps1 -ReportPath <report-json>`，并把 JSON artifact、校验结果和人工验收结果回填到 `docs/deployment/target-environment-acceptance.md`。
2. 最终验收前重跑完整验证链，并把命令结果写入阶段报告。
