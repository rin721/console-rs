# Quality

Read root `AGENTS.md` first.

Verification:

1. Backend changes: run `cargo fmt --all --check`, `cargo check --workspace`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo test --workspace`, and `cargo build --workspace`.
2. Route or contract changes: inspect `cargo run -p app -- routes --config configs/console.example.yaml` and ensure route registry, API catalog, permissions, and OpenAPI still derive from one source.
3. Frontend changes: run `npm --prefix web/app run typecheck`, `npm --prefix web/app run lint:i18n`, `npm --prefix web/app run build`, and Playwright for affected flows.
4. Security-sensitive changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/security-sensitive-scan.ps1`, and scan for raw tokens, secrets, password hashes, MFA secrets, and pending notification tokens in logs, URLs, localStorage/sessionStorage, tests, and snapshots.
5. Runtime-config boundary changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/runtime-config-boundary-scan.ps1` to prove frontend runtime headers, product code, client type, CSRF names, and API paths come from public settings or centralized API modules.
6. WebUI capability-boundary changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/webui-capability-boundary-scan.ps1` to prove System UI calls, including operation-record summaries, are backed by Rust route registry entries and old plugin/external deployment/bucket policy/mock production surfaces are absent.
7. Governance-sensitive changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/governance-scan.ps1` to keep internal engineering names neutral, old identities out of runtime contracts, plugin runtime contracts removed, and new schema free of plugin tables, then run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/goal-deliverable-audit.ps1` to prove the root workspace, required deliverables, migration matrix categories, project skills, route registry/OpenAPI evidence, WebUI i18n files, multi-dialect migrations, and reference-only Go boundary still match the active goal. When `aoi-admin/` source evidence or migration source scope changes, also run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/aoi-admin-source-audit.ps1` to prove old code, docs, route contracts, migrations, config, i18n, React, old rules, docs/ai, and plugin deletion evidence are still covered. When the stage acceptance report or final report structure changes, also run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/stage-acceptance-report-audit.ps1` to prove the required report sections, verification commands, residual-risk language, and below-100 progress boundary are still present.
8. Database schema changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/database-schema-dialect-scan.ps1` to ensure SQLite, PostgreSQL, and MySQL schema drafts expose the required platform tables without plugin runtime tables.
9. Database runtime changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/database-runtime-smoke.ps1` to prove plan, ping, insert-id-probe, migrate, setup-repository-probe, notification-repository-probe, system-repository-probe, migration-history, schema-check, database-preflight, and idempotent migrate reporting on a temporary SQLite runtime.
10. External database migration changes: run `scripts/database-external-smoke.ps1` against real PostgreSQL/MySQL services to prove migration, insert-id, SetupRepository, IamRepository, NotificationRepository, and SystemRepository probes, or rely on the CI `external-database-smoke` job when local services are unavailable.
11. Architecture-boundary changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/architecture-boundary-scan.ps1` to keep handlers away from infrastructure/SQL, service/usecase away from concrete database runtime, repository traits away from HTTP transport, `types` limited to app lifecycle contracts, and infrastructure capabilities injected through service/repository trait objects.
12. External database repository changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/repository-dialect-audit.ps1`; PostgreSQL/MySQL runtime changes must also pass `scripts/database-external-smoke.ps1` against real services or the CI service-container job.
13. Documentation changes: keep Chinese primary docs current and update the migration matrix when capability status changes.
14. Deployment-sensitive changes: run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/database-deploy-preflight.ps1 -ApplyMigrations` against the target or temporary deployment database to prove plan, ping, insert-id-probe, migrate, history, schema-check, and preflight, then run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/deployment-smoke.ps1` to prove the built WebUI is served by the Rust binary with health, readiness, OpenAPI, Setup, public settings, protected Prometheus scrape probes, and a local `target-environment-acceptance.ps1` report validated with `-AllowLocalHttp`. For real target releases, run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/target-environment-acceptance.ps1 -Driver <driver> -Url <database-url> -BaseUrl <base-url> -ApplyMigrations` with `CONSOLE_ACCEPTANCE_METRICS_SCRAPE_TOKEN` set in the process environment, archive the JSON report, then run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/validate-target-acceptance-report.ps1 -ReportPath <report-json>`. When the target report validator or evidence policy changes, also run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/target-acceptance-report-validator-smoke.ps1`. Only `scope=full` plus `result=passed` is final evidence, while `-AllowPartial` reports are diagnostics. The target report must include the `entrypoint-security` policy step proving non-local BaseUrl uses HTTPS, the `csrf-policy` HTTP step proving CSRF is enabled, the CSRF cookie is Secure and non-HttpOnly, and missing-CSRF non-GET `/api/*` is rejected, plus the `metrics-policy` HTTP step proving anonymous Prometheus metrics access is rejected with 401 or 403 without leaking metrics text, and configured Bearer scrape token access returns 200 with real metrics text without leaking raw token, API token, session token, or secret. `validate-target-acceptance-report.ps1 -AllowLocalHttp` is only for local smoke report structure checks and must not be used for final release evidence.
15. Before declaring the active `/goal` complete, run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/goal-completion-audit.ps1 -TargetAcceptanceReport "<report-json>" -RequireReady`; without `-RequireReady`, the script reports current readiness JSON and exits successfully for progress tracking.
16. Always finish with `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/trailing-whitespace-scan.ps1` and `git diff --check`.
