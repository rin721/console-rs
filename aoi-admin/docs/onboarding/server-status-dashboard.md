# 工作台服务器状态使用说明

服务器状态已合并到 `/admin` 工作台概览页，用于快速判断当前后端进程和主机资源是否健康。页面数据来自后端真实接口，不会展示前端伪造的 GPU、CI/CD 或服务进程明细。

## 进入页面先看哪里

1. 先看顶部摘要。健康、就绪、运行时长、API 数量和版本记录用于确认后台基础状态。
2. 再看服务器状态可视化。CPU、RAM 和最高磁盘使用率用环形图展示，便于快速扫视资源水位。
3. 最后看服务器监控趋势和 Go runtime 摘要。趋势历史来自后端短窗口采样，初次启动样本较少属于正常状态。

## 状态颜色含义

| 状态 | 含义 |
| --- | --- |
| 正常 | 指标处于健康范围。 |
| 关注 | 指标开始接近关注区间，可以继续观察。 |
| 预警 | 指标已经进入预警区间，应排查负载、容量或代码变化。 |
| 严重 | 指标进入高风险区间，应优先处理。 |
| 未知 | 当前接口没有返回可用数据。 |

## 每个区域怎么看

- CPU：展示后端快照中各核心使用率的平均值。
- 主机内存：展示 RAM 使用率以及已用/总量。
- 磁盘：展示后端返回挂载点中的最高使用率；没有磁盘样本时显示空状态。
- 服务器监控趋势：默认展示短窗口网络接收和发送速率，可切换到磁盘 IO 查看所有磁盘或单磁盘的读写 MB/s、读写次数/s 和平均 IO 延迟，数据来自 `/api/v1/system/server-metrics/history`。
- Go runtime、进程与构建：用于判断当前 Go 进程的运行时、内存、GC 和构建版本。

## 刷新说明

服务器快照和服务器历史会按后端采样间隔自动刷新；刷新按钮只用于立即重新读取工作台依赖的健康、就绪、服务器快照、服务器历史、API 清单和版本记录。当前 React 入口位于 `web/app/app/routes/admin/dashboard.tsx`，请求通过共享 API client 和 TanStack Query 执行。控件间距、状态颜色和文本必须使用 Aoi React token 与 i18n 资源，窄屏时依赖响应式布局而不是页面内临时断点。不要直接改页面代码里的接口路径、硬编码文案或后端未返回的指标。

## 新手提示

看到黄色或红色时，先看顶部异常摘要，再查看对应资源卡片。不要为了“让页面变绿”直接修改阈值；阈值应由维护者根据机器容量、业务峰值和运维策略在配置入口统一调整。
